Key Features

Pure Algorithm

Zero-dependency core implementation that works everywhere: CLI, WebAssembly, embedded systems.

Real-time Audio

Cross-platform audio I/O with low latency for live transmission and reception using malgo.

Browser Support

WebAssembly implementation allows FSK communication directly in web browsers.

Highly Configurable

Adjustable frequencies, FSK orders, baud rates. Support for audible and ultrasonic ranges.

Cross-platform

Works on Windows, macOS, Linux, BSD with support for various audio systems.

Multiple Applications

Device pairing, data transmission, retro computing, ultrasonic communication, IoT networks.

Modular Architecture

Clean separation of concerns enables code reuse across platforms and applications.

Core Package

fsk/core/

  • • Pure FSK algorithm
  • • No external dependencies
  • • WebAssembly compatible
  • • Signal processing logic

Realtime Package

fsk/realtime/

  • • Real-time audio I/O
  • • Cross-platform (malgo)
  • • Desktop/server only
  • • Chat and channels

Utils Package

fsk/utils/

  • • WAV file I/O
  • • Platform independent
  • • File operations
  • • Shared utilities

Usage Examples

Simple APIs for different use cases

Core Algorithm (WebAssembly Compatible)

import "github.com/gleicon/go-fsk/fsk/core"

// Create FSK modem
config := core.DefaultConfig()
modem := core.New(config)

// Encode message
signal := modem.Encode([]byte("Hello FSK"))

// Decode signal
decoded := modem.Decode(signal)
fmt.Printf("Decoded: %s\n", string(decoded))

Real-time Audio (Desktop/Server)

import (
    "github.com/gleicon/go-fsk/fsk/core"
    "github.com/gleicon/go-fsk/fsk/realtime"
)

modem := core.New(core.DefaultConfig())

// Real-time transmission
transmitter, _ := realtime.NewTransmitter(modem)
transmitter.Transmit([]byte("Live data"))
